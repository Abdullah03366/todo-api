# Dag 4 – Backend uitbreiden + SSW/security

## Doel

Security en robuustheid.

## Output

Veilige, uitgebreide API.

## Opdracht

1. Implementeer veiligheid:
   - Inputvalidatie

2. Voeg unit tests toe

3. Beschrijf met welke extra veiligheidszaken rekening gehouden moet worden

4. **OPTIONEEL: Maak een eenvoudige frontend** (HTML/CSS/JavaScript of React) die communiceert met je API
   - Toon alle todos in een lijst
   - Voeg POST met een form toe
   - Voeg een delete-knop per todo toe
   - Dit is niet verplicht, maar geeft extra inzicht in hoe je de API gebruikt.

## Leerdoelen

- Security-first coding
- Testen
- API uitbreiden
- (Optioneel) Frontend integratie met backend

## Voorbeelden

Zie [Dag 4 – Code Voorbeelden (Tests & Security)](./voorbeelden/Dag4_Voorbeelden.md) voor unit tests en een optionele frontend.

## Eindpresentatie (Dag 4)

Presenteer de veilige en uitgebreide backend:

- **Input Validation**: Welke validaties heb je toegevoegd? (type, length, sanitization)
- **Unit Tests**: Hoeveel tests heb je geschreven? Welke test coverage heb je bereikt?
- **Security Measures**: SQL injection, XSS, CSRF preventie - hoe zit het?
- **Optional Features**: Heb je een PUT-endpoint of extra beveiligingsfeatures geïmplementeerd?
- **Code Review**: Welke beveiligingsprincipes heb je toegepast (OWASP, SSW guidelines)?
- **Lessons Learned**: Wat was het lastigste deel? Wat zou je volgende keer anders doen?
