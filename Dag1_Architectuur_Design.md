# Dag 1 – Architectuur & Design (alle tech stacks)

## Doel

Het begrijpen, ontwerpen, documenteren en uitleggen.

## Output

Een presentatie van 10 minuten + architectuurdocument.

## Opdracht

1. Maak een high-level architectuurontwerp van een TODO-app met:
   - Backend API
   - Azure resources
   - CI/CD pipeline (GitHub Actions)

2. Lever een diagram op (bijv. met Draw.io / Mermaid)

3. Beschrijf:
   - API endpoints
   - Datamodel
   - Benodigde Azure services
   - Security-strategie (SSW principles)

## Voorbeelden

Zie [Dag 1 – Code Voorbeelden](./voorbeelden/Dag1_Voorbeelden.md) voor een architectuurdiagram en API-endpoints.

## Leerdoelen

- Architectuur kunnen uitleggen
- Overbrengen van informatie
- API-first ontwerpen
- Cloud & security bewustzijn

## Eindpresentatie (Dag 1)

Presenteer het architectuurontwerp van 10 minuten:

- **Architectuurdiagram**: Toon het diagram met alle componenten (API, Azure services, CI/CD)
- **API Endpoints**: Leg de endpoints en versioning uit
- **Datamodel**: Beschrijf de databaseschema's en relaties
- **Azure Services**: Welke services heb je gekozen en waarom?
- **Security**: Hoe zit het met SSW principles en security in je design?
- **Vragen**: Wat heb je geleerd? Welke uitdagingen zijn er?
