# Dag 2 – Pipelines bouwen (GitHub Actions)

## Doel

Een werkende CI-pipeline + build-artifact.

## Output

Pipeline YAML + screenshots + korte demo.

## Opdracht

1. Maak een simpele pipeline die:
   - Code bouwt (.NET → dotnet build, Java → mvn package)
   - Tests draait
   - Een build-artifact publiceert

2. Bonus: SonarQube / CodeQL integreren voor SSW/security.

## Voorbeelden

Zie [Dag 2 – Code Voorbeelden (CI/CD Pipelines)](./voorbeelden/Dag2_Voorbeelden.md) voor GitHub Actions-setup en best practices.

## Leerdoelen

- CI/CD basis
- Build pipelines begrijpen
- Security en code quality checks

## Eindpresentatie (Dag 2)

Demonstreer de werkende CI-pipeline:

- **Pipeline YAML**: Toon de structuur en configuratie van je GitHub Actions-workflow
- **Build Process**: Laat zien hoe de code gebuild wordt (.NET/Java)
- **Artifacts**: Waar worden artifacts opgeslagen? Hoe kunnen ze gebruikt worden?
- **Learnings**: Wat was moeilijk? Hoe heb je fouten opgelost?
